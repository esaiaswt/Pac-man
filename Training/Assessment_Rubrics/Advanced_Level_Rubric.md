# Advanced Level Assessment Rubric

**Modules 7-9: OOP, AI, Audio APIs**

## Evaluation Criteria

### Technical Mastery (70%)

#### Object-Oriented Programming (25%)
| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **Classes** | Expert class design | Can create classes | Class errors common | Cannot use classes |
| **Inheritance** | Effective use of inheritance | Basic inheritance works | Inheritance issues | No inheritance |
| **Encapsulation** | Properly encapsulated code | Some encapsulation | Minimal encapsulation | No encapsulation |
| **Polymorphism** | Advanced polymorphic design | Basic polymorphism | Limited polymorphism | No polymorphism |

#### Game AI (25%)
| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **Pathfinding** | Sophisticated algorithms | Basic pathfinding works | Simple pathfinding | No pathfinding |
| **Behavior Design** | Unique, interesting AI | Functional AI | Basic AI | No AI implementation |
| **State Machines** | Complex state systems | Basic states work | State bugs | No state machine |
| **Algorithm Efficiency** | Optimized algorithms | Decent performance | Performance issues | Very inefficient |

#### Audio APIs (20%)
| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **Web Audio API** | Advanced audio techniques | Can generate sounds | Basic sounds only | Cannot use audio API |
| **Sound Design** | Professional sound effects | Decent sound effects | Basic sounds | Poor/no sounds |
| **Audio Management** | Clean audio system | Working audio | Audio bugs | Broken audio |

### Advanced Concepts (20%)

| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **Code Architecture** | Excellent architecture | Good structure | Some structure | Poor architecture |
| **Performance** | Optimized, smooth | Good performance | Some lag | Poor performance |
| **Best Practices** | Follows all best practices | Most best practices | Some best practices | No best practices |

### Innovation & Creativity (10%)

| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **Original Ideas** | Highly innovative | Some creativity | Limited creativity | No creativity |
| **Problem Solving** | Novel solutions | Standard solutions | Simple solutions | Cannot solve problems |

## Capstone Project Requirements

Students must complete at least ONE of the following:

1. **Complete Challenge 2 (Custom Ghost)** with unique AI
2. **Complete Challenge 3 (Power-Ups)** with 3+ power-ups
3. **Original game** using OOP and AI principles
4. **Major Pac-Man enhancement** (new features, levels, etc.)

Plus:
- Technical documentation
- Code comments explaining complex logic
- Demo video or presentation
- Reflection essay on learning journey

## Industry-Ready Criteria

For students aiming for professional development:
- 90%+ on all assessments
- Completed all 5 challenges
- Clean, documented code
- Portfolio-quality project
- Can explain technical decisions
