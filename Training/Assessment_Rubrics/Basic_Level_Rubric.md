# Basic Level Assessment Rubric

**Modules 1-3: Introduction, HTML, CSS**

## Evaluation Criteria

### Technical Skills (60%)

#### HTML Knowledge (20%)
| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **Document Structure** | Perfect HTML5 structure with all required elements | Mostly correct structure with minor errors | Some structural issues | Many structural problems |
| **Element Usage** | Uses appropriate semantic elements consistently | Uses mostly appropriate elements | Limited element variety | Incorrect element usage |
| **Attributes** | Correctly uses IDs, classes, and other attributes | Mostly correct attribute usage | Some attribute errors | Frequent attribute mistakes |

#### CSS Knowledge (20%)
| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **Selectors** | Uses variety of selectors appropriately | Uses basic selectors correctly | Limited selector knowledge | Selector errors common |
| **Styling** | Creates attractive, well-styled pages | Good styling with minor issues | Basic styling, needs improvement | Minimal or poor styling |
| **Layout** | Effective use of box model and flexbox | Decent layout with some issues | Layout problems | Cannot create layouts |

#### Tools & Environment (20%)
| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **DevTools** | Proficiently uses DevTools for debugging | Can use basic DevTools features | Minimal DevTools usage | Cannot use DevTools |
| **File Management** | Well-organized project structure | Acceptable organization | Some organization issues | Poor file management |
| **Code Quality** | Clean, well-formatted HTML/CSS | Mostly clean code | Some formatting issues | Poorly formatted code |

### Problem Solving & Application (25%)

| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **Worksheet Completion** | All tasks completed correctly | Most tasks completed well | Half of tasks completed | Minimal completion |
| **Challenges** | Completed bonus challenges | Completed required challenges | Attempted challenges | Did not attempt |
| **Creativity** | Highly creative solutions | Some creativity shown | Limited creativity | No creativity |

### Understanding & Communication (15%)

| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **Concept Comprehension** | Deep understanding of all concepts | Good understanding | Partial understanding | Minimal understanding |
| **Technical Vocabulary** | Uses terms accurately | Mostly accurate terminology | Some correct terms | Incorrect terminology |
| **Self-Assessment** | Accurate self-evaluation | Reasonable self-assessment | Somewhat inaccurate | No self-awareness |

---

## Grading Scale

| Total Points | Letter Grade | Assessment |
|--------------|--------------|------------|
| 90-100 | A | Excellent - Ready for Intermediate |
| 80-89 | B | Good - Ready with review |
| 70-79 | C | Satisfactory - Needs more practice |
| 60-69 | D | Developing - Significant gaps |
| Below 60 | F | Not ready - Repeat modules |

---

## Portfolio Requirements

Students must submit:

1. **Completed Worksheets** for all 3 modules
2. **At least 3 HTML pages** created from scratch
3. **At least one styled page** showcasing CSS skills
4. **Screenshot evidence** of using DevTools
5. **Written reflection** (1 paragraph) on what they learned

---

## Instructor Notes

### Passing Requirements
- Minimum 70% overall
- Completed all required worksheet tasks
- Can explain HTML structure and CSS basics
- Can use browser DevTools

### Red Flags
- Cannot create basic HTML structure
- Doesn't understand selectors
- Cannot debug with DevTools
- Copies code without understanding

### Remediation
If student scores below 70%:
1. Review specific weak areas
2. Provide additional practice worksheets
3. One-on-one tutoring session
4. Retake assessment before advancing
