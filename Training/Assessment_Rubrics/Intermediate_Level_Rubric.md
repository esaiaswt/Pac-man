# Intermediate Level Assessment Rubric

**Modules 4-6: JavaScript, Game Development, Interactivity**

## Evaluation Criteria

### Technical Skills (65%)

#### JavaScript Fundamentals (25%)
| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **Syntax** | Perfect syntax, no errors | Minor syntax issues | Frequent syntax errors | Cannot write basic code |
| **Variables & Data Types** | Expert use of variables and types | Good understanding | Some confusion | Minimal understanding |
| **Functions** | Creates reusable, well-structured functions | Can write basic functions | Function errors common | Cannot write functions |
| **Control Flow** | Effective use of conditionals and loops | Basic control flow works | Logic errors frequent | Cannot implement logic |
| **DOM Manipulation** | Fluent DOM interaction | Can manipulate DOM | DOM errors common | Cannot use DOM |

#### Game Development (20%)
| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **Canvas API** | Advanced canvas techniques | Can draw basic shapes | Canvas errors | Cannot use canvas |
| **Game Loop** | Smooth, efficient game loop | Working game loop | Loop issues | Cannot implement loop |
| **Collision Detection** | Multiple collision types | Basic collision works | Collision bugs | No collision detection |
| **Animation** | Smooth, professional animation | Basic animation works | Choppy animation | No animation |

#### Interactivity (20%)
| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **Event Handling** | Complex event systems | Basic events work | Event bugs | Cannot handle events |
| **State Management** | Clean state transitions | Basic states work | State bugs | No state management |
| **Data Persistence** | Effective use of localStorage | Can save/load data | Storage issues | No persistence |

### Problem Solving (20%)

| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **Debugging** | Independently debugs complex issues | Can debug basic problems | Needs help debugging | Cannot debug |
| **Code Organization** | Excellent structure and organization | Decent organization | Some organization | Poor organization |
| **Algorithm Design** | Creates efficient algorithms | Working algorithms | Inefficient solutions | Cannot design algorithms |

### Understanding (15%)

| Criteria | Exemplary (4) | Proficient (3) | Developing (2) | Beginning (1) |
|----------|---------------|----------------|----------------|---------------|
| **Concept Mastery** | Deep understanding of all concepts | Good understanding | Partial understanding | Minimal understanding |
| **Code Reading** | Can analyze complex code | Can read basic code | Struggles with code | Cannot read code |

---

## Grading Scale

| Total Points | Grade | Status |
|--------------|-------|--------|
| 90-100 | A | Excellent - Ready for Advanced |
| 80-89 | B | Good - Ready with review |
| 70-79 | C | Satisfactory - Practice needed |
| 60-69 | D | Developing - Gaps exist |
| Below 60 | F | Not ready - Repeat modules |

---

## Portfolio Requirements

1. **Working game** with canvas, controls, and score tracking
2. **Completed worksheets** for modules 4-6
3. **Code samples** demonstrating key concepts
4. **Game design document** explaining choices
5. **Video demo** or screenshots of working game

---

## Passing Requirements
- Minimum 70% overall
- Working interactive game
- Can write functions and handle events
- Understands game loops and collision
