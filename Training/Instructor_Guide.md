# Instructor Guide: PAC-MAN Coding Training Program

## üìö Program Overview

This comprehensive training program uses the Pac-Man game to teach web development from basic HTML/CSS to advanced JavaScript concepts including OOP and AI. Total duration: 40-60 hours across 9 modules.

---

## üéØ Teaching Philosophy

### Key Principles

1. **Project-Based Learning**: All concepts tied to real game code
2. **Scaffolded Instruction**: Build gradually from simple to complex
3. **Hands-On Practice**: Every lesson includes practical exercises
4. **Self-Paced**: Materials support independent learning
5. **Engagement First**: Make learning fun and rewarding

### Student-Centered Approach

- Encourage experimentation and "productive failure"
- Celebrate small wins and progress
- Foster peer collaboration
- Support different learning paces
- Provide multiple pathways to mastery

---

## üìÖ Suggested Schedules

### Intensive Bootcamp (2 Weeks)
- **Week 1**: Modules 1-6 (Basic + Intermediate)
- **Week 2**: Modules 7-9 (Advanced) + Challenges

### Semester Course (12-16 Weeks)
- **Weeks 1-4**: Modules 1-3 (Basic)

### After-School Club (6 Months)
- 2 hours per week, 1-2 modules per month
- More time for exploration and challenges
- Group projects and peer learning

## üóÇÔ∏è Module-by-Module Guide

### Module 1: Introduction to Web Development

**Key Learning Goals**:
- Understand web technology roles
- Set up development environment
- Use browser DevTools

**Common Student Challenges**:
- Installing VS Code (provide step-by-step)
- Finding files and folders (Windows vs Mac)
- Understanding DevTools (live demonstration helps)

**Teaching Tips**:
- Do live coding demos
- Show your own DevTools workflow
- Let students explore on their own
- Pair students for peer support

**Time Allocation**:
- Lesson: 45 min
- Worksheet: 60-90 min
- Q&A: 30 min

**Assessment**:
- Can open Pac-Man game
- Can open and navigate DevTools
- Can explain HTML/CSS/JS roles

---

### Module 2: HTML Basics

**Key Learning Goals**:
- Write proper HTML structure
- Use common elements
- Understand attributes

**Common Student Challenges**:
- Forgetting closing tags
- Mixing up `<div>` and `<span>`
- ID vs class confusion

**Teaching Tips**:
- Use color-coding in editor
- Show live preview alongside code
- Practice with interactive tools (CodePen)
- Emphasize semantic HTML early

**Debugging Help**:
- Missing closing tags ‚Üí use DevTools Elements tab
- Attribute errors ‚Üí check quotes and spacing
- Validation ‚Üí use W3C validator

**Time Allocation**:
- Lesson: 90-120 min
- Worksheet: 120-150 min
- Practice: 60-90 min

---

### Module 3: CSS Styling Fundamentals

**Key Learning Goals**:
- CSS selectors and specificity
- Box model understanding
- Layout with flexbox
- Animations and effects

**Common Student Challenges**:
- Specificity confusion
- Box model (padding vs margin)
- Flexbox alignment
- Cascade and inheritance

**Teaching Tips**:
- Draw box model diagrams on whiteboard
- Use Flexbox Froggy game for practice
- Show before/after examples
- Encourage experimentation with values

**Best Practices to Emphasize**:
- Use external stylesheets
- Organize CSS logically
- Use CSS variables for consistency
- Comment your code

**Time Allocation**:
- Lesson: 120-150 min
- Worksheet: 150-210 min
- Creative projects: 90 min

---

### Module 4: JavaScript Fundamentals

**Key Learning Goals**:
- Variables and data types
- Functions and scope
- Control flow
- DOM manipulation

**Common Student Challenges**:
- Syntax errors (missing semicolons, quotes)
- Variable scope confusion
- Understanding functions vs methods
- Timing of DOM manipulation

**Teaching Tips**:
- Start with console.log extensively
- Use debugger and breakpoints early
- Show error messages and how to read them
- Practice with small, isolated examples

**Live Coding Ideas**:
- Build a click counter
- Create a color changer
- Make a simple calculator
- Build a to-do list

**Time Allocation**:
- Lesson: 120-150 min
- Worksheet: 180-240 min
- Mini-projects: 90-120 min

---

### Module 5: Game Development Basics

**Key Learning Goals**:
- Canvas API fundamentals
- Game loop implementation
- Animation techniques
- Collision detection

**Common Student Challenges**:
- Understanding coordinate systems
- Game loop timing
- Collision math
- Performance optimization

**Teaching Tips**:
- Draw coordinate system diagrams
- Start with static shapes before animation
- Use console.log to debug positions
- Show requestAnimationFrame vs setInterval

**Milestone Checks**:
- Can draw shapes on canvas
- Game loop running smoothly
- Object moving on screen
- Basic collision working

**Time Allocation**:
- Lesson: 120-150 min
- Worksheet: 180-240 min
- Game building: 120-180 min

**Teaching Tips**:

### Module 7: Object-Oriented Programming

**Key Learning Goals**:
- Classes and objects
- Inheritance
- Encapsulation
- Polymorphism

**Common Student Challenges**:
- `this` keyword confusion
- Constructor vs regular methods
- When to use inheritance
- Abstract vs concrete

**Teaching Tips**:
- Use real-world analogies (Car class, etc.)
- Draw UML diagrams
- Show Pac-Man ghost hierarchy
- Practice refactoring procedural code to OOP

**Conceptual Breakthroughs**:
- "Classes are blueprints"
- "Inheritance is about 'is-a' relationships"
- "Encapsulation hides complexity"

**Time Allocation**:
- Lesson: 150-180 min
- Worksheet: 180-240 min
- Refactoring exercises: 120 min

## üîß Technical Setup Guide

### Required Software
1. **Text Editor**: VS Code (recommended)
   - Extensions: Live Server, Prettier, ESLint
2. **Browser**: Chrome or Firefox (for DevTools)
3. **Version Control** (Optional): Git

### Classroom Setup
- Ensure all students can access files
- Test network/internet connectivity
- Have backup USB drives with materials
- Prepare troubleshooting guide

### Common Technical Issues

**Issue**: Game doesn't load
- **Fix**: Check file paths, console errors

**Issue**: CSS not applying  
- **Fix**: Verify link tag, hard refresh (Ctrl+F5)

**Issue**: JavaScript errors
- **Fix**: Open console, read error messages

**Issue**: Canvas is blank
- **Fix**: Check canvas dimensions, context setup
